## 1. Ring Buffer vs Perf Buffer

Both are **kernel-to-userspace** data channels. Ring buffer is the newer,
preferred one. A user ring buffer (reverse direction) also exists.

### 1.1 Architecture Comparison

```
  BPF Program (kernel)              Userspace
  ────────────────────              ─────────

  Perf Buffer (per-CPU, older):
  ┌──────────┐ ┌──────────┐
  │ CPU 0 buf│ │ CPU 1 buf│ ...    each CPU has own ring
  └─────┬────┘ └─────┬────┘        userspace polls ALL of them
        │            │              event ordering NOT guaranteed
        └────────────┘              wakeup per event (expensive)
              │
     bpf_perf_event_output()

  Ring Buffer (shared, newer):
  ┌─────────────────────────┐
  │   Single shared ring    │      all CPUs write to ONE ring
  │   (lock-free MPSC)      │      event ordering preserved
  │   ┌───┬───┬───┬───┐    │      userspace polls ONE fd
  │   │ev1│ev2│ev3│   │    │      wakeup batching (cheap)
  │   └───┴───┴───┴───┘    │
  └────────────┬────────────┘
               │
     bpf_ringbuf_output()
     bpf_ringbuf_reserve() + submit()
```

### 1.2 Perf Buffer Usage

```c
/* BPF side */
struct {
    __uint(type, BPF_MAP_TYPE_PERF_EVENT_ARRAY);
    __uint(key_size, sizeof(int));
    __uint(value_size, sizeof(int));
} events SEC(".maps");

SEC("tracepoint/syscalls/sys_enter_write")
int trace_write(struct trace_event_raw_sys_enter *ctx) {
    struct event e = { .pid = bpf_get_current_pid_tgid() >> 32 };

    /* Copy data to perf buffer — always copies */
    bpf_perf_event_output(ctx, &events, BPF_F_CURRENT_CPU,
                          &e, sizeof(e));
    return 0;
}

/* Userspace side */
perf_buffer__poll(pb, timeout_ms);  /* polls all per-CPU buffers */
```

**Properties:**
- One ring per CPU — no cross-CPU contention
- `bpf_perf_event_output()` always copies data
- If per-CPU buffer fills up, events are **dropped** (no backpressure)
- Userspace must `epoll` on N file descriptors (one per CPU)
- Event ordering only within same CPU

### 1.3 Ring Buffer Usage

```c
/* BPF side */
struct {
    __uint(type, BPF_MAP_TYPE_RINGBUF);
    __uint(max_entries, 256 * 1024);  /* 256KB shared */
} rb SEC(".maps");

SEC("tracepoint/syscalls/sys_enter_write")
int trace_write(void *ctx) {
    struct event e = { .pid = bpf_get_current_pid_tgid() >> 32 };

    /* Option 1: copy (simple) */
    bpf_ringbuf_output(&rb, &e, sizeof(e), 0);

    /* Option 2: reserve + fill + submit (zero-copy) */
    struct event *re = bpf_ringbuf_reserve(&rb, sizeof(*re), 0);
    if (!re) return 0;
    re->pid = bpf_get_current_pid_tgid() >> 32;
    bpf_ringbuf_submit(re, 0);  /* or bpf_ringbuf_discard(re, 0) */

    return 0;
}

/* Userspace side */
ring_buffer__poll(rb, timeout_ms);  /* polls ONE fd */
```

> **Size constraint:** `bpf_ringbuf_reserve()` requires a **compile-time
> constant** size (`ARG_CONST_ALLOC_SIZE_OR_ZERO` — verifier checks
> `tnum_is_const()`). For variable-sized events, use `bpf_ringbuf_output()`
> (copies, runtime size OK) or `bpf_ringbuf_reserve_dynptr()` (zero-copy,
> runtime size OK via `ARG_ANYTHING`).
>
> | Function | Size must be constant? | Zero-copy? |
> |----------|----------------------|------------|
> | `bpf_ringbuf_reserve` | **Yes** (compile-time) | Yes |
> | `bpf_ringbuf_output` | No (runtime OK) | No (copies) |
> | `bpf_ringbuf_reserve_dynptr` | No (runtime OK) | Yes |

**Properties:**
- Single shared ring — all CPUs write to same buffer
- Lock-free multi-producer single-consumer (MPSC)
- `reserve + submit` is zero-copy (write directly to ring)
- `output` always copies (simpler but slower)
- Global event ordering preserved
- Userspace polls ONE file descriptor
- Wakeup batching via `BPF_RB_NO_WAKEUP` / `BPF_RB_FORCE_WAKEUP` flags

### 1.4 Feature Comparison

| Feature | Perf Buffer | Ring Buffer |
|---------|-------------|-------------|
| Introduced | v4.15 | v5.8 |
| Topology | Per-CPU rings | Single shared ring |
| Memory efficiency | Wastes memory on idle CPUs | Shared pool — no waste |
| Zero-copy write | No (always copies) | Yes (`reserve + submit`) |
| Event ordering | Per-CPU only | Global |
| Userspace polling | N fds (one per CPU) | 1 fd |
| Wakeup control | Per-event wakeup | Batched/manual |
| Backpressure | Drop on overflow | `reserve` returns NULL |
| Dynptr support | No | Yes (`bpf_ringbuf_reserve_dynptr`) |

## 2. Wakeup Control

When BPF writes to the ring buffer, the kernel decides whether to wake up
the userspace consumer (blocked in `ring_buffer__poll()`). These flags
control that decision:

| Flag | Wakeup consumer? | Use case |
|------|-----------------|----------|
| `0` (default) | Adaptive (if idle) | General purpose |
| `BPF_RB_NO_WAKEUP` | Never | High-frequency batching |
| `BPF_RB_FORCE_WAKEUP` | Always | Latency-critical events |

**Typical batch + flush pattern:**

```c
SEC("tracepoint/...")
int trace(void *ctx) {
    /* High-frequency events — no wakeup, let them batch */
    bpf_ringbuf_output(&rb, &event, sizeof(event), BPF_RB_NO_WAKEUP);

    /* Every Nth event or on special condition — force wakeup */
    if (should_flush())
        bpf_ringbuf_output(&rb, &flush_marker, sizeof(flush_marker),
                           BPF_RB_FORCE_WAKEUP);
    return 0;
}
```

This reduces wakeup overhead from one-per-event to one-per-batch —
significant for high-throughput tracing (millions of events/sec).

### 2.1 Userspace Consumption Patterns

The userspace side has two approaches for reading events:

**`ring_buffer__poll()` — event-driven with timeout:**

```c
while (1) {
    ring_buffer__poll(rb, 100 /* ms timeout */);
    /* Uses epoll_wait() internally.
     * With BPF_RB_NO_WAKEUP: wakes only on timeout (100ms latency).
     * With BPF_RB_FORCE_WAKEUP: wakes immediately on event.
     */
}
```

**`ring_buffer__consume()` — busy-poll, no syscall:**

```c
while (1) {
    int n = ring_buffer__consume(rb);
    /* Directly reads shared memory — no epoll, no sleep.
     * Returns immediately with count of consumed events.
     * Perfect with BPF_RB_NO_WAKEUP — no wakeup needed.
     */
}
```

### 2.2 Consumption Pattern Comparison

| Pattern | Wakeup flags | CPU usage | Latency |
|---------|-------------|-----------|---------|
| `poll(100ms)` | NO_WAKEUP | Low (sleeps) | Up to 100ms |
| `poll(1ms)` | NO_WAKEUP | Medium | Up to 1ms |
| `consume()` spin | NO_WAKEUP | 100% one core | Sub-microsecond |
| `poll(infinity)` | FORCE_WAKEUP | Minimal | Event-driven |

For io_uring/ublk style workloads where you're already busy-polling,
`ring_buffer__consume()` + `BPF_RB_NO_WAKEUP` is the natural fit.

## 3. Ring Buffer Internals

### 3.1 Shared Memory Layout

```
  mmap'd by userspace via bpf map fd:

  Page 0 (read-write by userspace):
  ┌──────────────────────────────────────┐
  │  consumer_pos  (userspace advances)  │
  ├──────────────────────────────────────┤

  Page 1 (read-only to userspace):
  │  producer_pos  (kernel advances)     │
  ├──────────────────────────────────────┤

  Pages 2+ (data ring, double-mapped):
  │  data pages (power-of-2 size)        │
  │  ┌────┬────┬────┬────┬──────┐       │
  │  │hdr1│data│hdr2│data│ free │       │
  │  └────┴────┴────┴────┴──────┘       │
  └──────────────────────────────────────┘
       ↑                    ↑
    consumer_pos         producer_pos
```

### 3.2 Event Header Format

Each event in the ring has an 8-byte header:

```c
struct bpf_ringbuf_hdr {
    __u32 len;      /* data length (lower 28 bits)
                     * bit 31: BPF_RINGBUF_BUSY_BIT  -- being written
                     * bit 30: BPF_RINGBUF_DISCARD_BIT -- discarded
                     */
    __u32 pg_off;   /* page offset for this record */
};
```

### 3.3 BUSY and DISCARD Bits

```
  BPF side (reserve + submit):

  1. bpf_ringbuf_reserve():
     hdr.len = size | BPF_RINGBUF_BUSY_BIT
     producer_pos advanced
     BPF program writes data into reserved space

  2. bpf_ringbuf_submit():
     clear BUSY bit -> consumer can now read

  2b. bpf_ringbuf_discard() instead:
     set DISCARD bit -> consumer skips this entry

  Consumer behavior:
  - Sees BUSY bit -> spins waiting for it to clear (writer in progress)
  - Sees DISCARD bit -> skips entry, advances consumer_pos
  - Neither bit -> valid event, calls user callback
```

### 3.4 The Double-Mapping Trick

The data pages are mapped **twice** contiguously in virtual address space
so wrap-around events can be read as contiguous memory:

```
  Physical pages:     [P0] [P1] [P2] [P3]

  Virtual mapping:    [P0] [P1] [P2] [P3] [P0] [P1] [P2] [P3]
                      |-- first copy ----|---- second copy ---|

  Event that wraps around the physical end:
                                    |----------|
                              [P2] [P3  P0] [P1]
                                    contiguous read!
```

Without double-mapping, consuming a wrap-around event would need two
separate reads. With it, any event at any position is a contiguous range.

### 3.5 Lock-Free Consume Algorithm

```c
/* libbpf: ringbuf_process_ring() */
ringbuf_process_ring(ring):
    cons_pos = smp_load_acquire(ring->consumer_pos)
    prod_pos = smp_load_acquire(ring->producer_pos)

    while (cons_pos < prod_pos):
        hdr = ring->data + (cons_pos & ring->mask)
        len = smp_load_acquire(&hdr->len)

        if (len & BPF_RINGBUF_BUSY_BIT):
            break    /* writer in progress, stop here */

        cons_pos += roundup(8 + (len & ~DISCARD_BIT), 8)

        if (!(len & BPF_RINGBUF_DISCARD_BIT)):
            ring->sample_cb(ring->ctx, hdr->data, len)

    smp_store_release(ring->consumer_pos, cons_pos)
    return count
```

Everything is lock-free. The only synchronization is
`smp_load_acquire` / `smp_store_release` on the position counters.

## 4. Wakeup Mechanism: How Userspace Gets Notified

### 4.1 Kernel Data Structures

```c
/* kernel/bpf/ringbuf.c */
struct bpf_ringbuf {
    wait_queue_head_t   waitq;     /* userspace sleeps here */
    struct irq_work     work;      /* deferred wakeup */
    unsigned long       consumer_pos;
    unsigned long       producer_pos;
    char                data[];    /* event data pages */
};
```

### 4.2 Why `irq_work` Instead of Direct `wake_up`

BPF programs run in contexts where `wake_up()` can't be called directly:

| Context BPF runs in | Can call wake_up? | Solution |
|---------------------|-------------------|----------|
| Softirq | Risky | irq_work defers safely |
| Hard IRQ | No | irq_work defers to softirq |
| NMI (perf events) | Absolutely not | irq_work defers to safe point |

`irq_work_queue()` is always safe — it just sets a per-CPU flag. The
actual `wake_up_all()` runs when the CPU reaches a safe context.

### 4.3 Wakeup Decision Logic

```c
/* kernel/bpf/ringbuf.c -- after submit/output */
static void bpf_ringbuf_commit(void *sample, u64 flags, bool discard)
{
    /* Clear BUSY bit (and maybe set DISCARD bit) */
    xchg(&hdr->len, new_len);

    if (flags & BPF_RB_NO_WAKEUP)
        return;                        /* never wake */

    if (flags & BPF_RB_FORCE_WAKEUP)
        goto wake;                     /* always wake */

    /* Default: adaptive -- wake only if consumer was idle */
    if (consumer_pos == producer_pos_before_this_event)
        goto wake;                     /* consumer caught up -> wake */
    return;                            /* consumer behind -> skip */

wake:
    irq_work_queue(&rb->work);         /* deferred wakeup */
}
```

### 4.4 How `epoll` Connects to the Waitqueue

```
  ring_buffer__new(map_fd, callback):
    1. mmap the ring buffer (consumer/producer/data pages)
    2. epoll_fd = epoll_create1()
    3. epoll_ctl(epoll_fd, ADD, map_fd, EPOLLIN)
       -> kernel: bpf_ringbuf_map_poll()
          -> poll_wait(file, &rb->waitq, ...)
             (registers fd on rb->waitq)

  ring_buffer__poll(rb, timeout):
    1. epoll_wait(epoll_fd, ..., timeout)
       -> sleeps on rb->waitq
       -> woken by wake_up_all() from irq_work
    2. ringbuf_process_ring()  -- consume events
```

### 4.5 Full Wakeup Flow

```
  BPF program               Kernel                      Userspace
  (any context)              (irq_work)
  ────────────               ──────────                  ─────────

  bpf_ringbuf_submit()
       |
       +- clear BUSY bit
       +- advance producer_pos
       |
       +- consumer was idle?
       |   YES: irq_work_queue()
       |   NO:  return (skip)
       |
       |                     irq_work fires:
       |                     bpf_ringbuf_notify()
       |                          |
       |                     wake_up_all(&rb->waitq)
       |                          |
       |                          +---->  epoll_wait() returns
       |                                       |
       |                                  ringbuf_process_ring()
       |                                       |
       |                                  read events, call callback
       |                                  advance consumer_pos
       |                                       |
       |                                  epoll_wait() again (sleep)
```

## 5. When to Use Which

- **Ring buffer** — default choice for new code. Better in almost every way.
- **Perf buffer** — only if you need per-CPU isolation (avoid cross-CPU cache
  bouncing in extreme throughput scenarios), or targeting old kernels (< v5.8).

## 6. User Ring Buffer (v6.1) — Reverse Direction

Userspace writes, BPF program reads:

```c
struct {
    __uint(type, BPF_MAP_TYPE_USER_RINGBUF);
    __uint(max_entries, 256 * 1024);
} urb SEC(".maps");

/* BPF side — drain messages from userspace */
static long handle_msg(struct bpf_dynptr *dynptr, void *ctx) {
    /* read user-submitted data via dynptr */
    return 0;
}

SEC("tracepoint/...")
int drain(void *ctx) {
    bpf_user_ringbuf_drain(&urb, handle_msg, NULL, 0);
    return 0;
}
```

```
  Direction summary:
  ┌────────────────┐       ┌───────────┐
  │  BPF program   │──────→│ Userspace │  ringbuf (kernel → user)
  │  (kernel)      │←──────│           │  user_ringbuf (user → kernel)
  └────────────────┘       └───────────┘
```
